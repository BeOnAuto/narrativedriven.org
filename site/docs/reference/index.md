# Reference

Look things up. These pages are designed for quick access when you need to check a definition, understand a construct, or review the model structure.

NDD organises every system into four levels: **Domain → Narrative → Scene → Moment**. The references below explain each one, the moment types, and the supporting constructs.

**[Glossary](/reference/glossary)**
Every NDD term defined. The four-level hierarchy, moments and their types, actors, entities, specs, event model constructs, and key principles.

**[Moment Types](/reference/moment-types)**
The four moment types (command, query, react, experience) with full explanations, spec patterns, and examples from the Concert Booking domain.

**Model Reference** *(coming soon)*
The complete NDD model schema. Every construct, every field, every relationship. The technical reference for developers working with the Zod schema or JSON model directly.
