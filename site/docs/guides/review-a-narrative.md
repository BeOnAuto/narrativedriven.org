# Review a Narrative

![Narrative card with magnifying glass, green checkmarks, a red X, and a red question mark beside specific lines](/images/heroes/review-a-narrative.png){.page-hero}

A narrative is only valuable if humans can review it before build.

Use this checklist.

## 1. Review the goal

Ask:

- Is this goal real?
- Is it too broad?
- Is it too small?
- Does it belong in this capability?

Bad:

> Manage everything

Better:

> Submit team timesheets

## 2. Review the outcomes

Ask:

- What becomes true?
- Is each outcome independently understandable?
- Are any outcomes missing?
- Are failure outcomes represented when they change the actor's state?

Examples:

- Timesheet submitted
- Submission blocked for missing hours
- Entry validated
- Period closed

## 3. Review the moments

Ask:

- Does the flow make sense?
- Is anything skipped?
- Are user and system moments separated?
- Are automatic system responses explicit?
- Does each moment have the right type?

## 4. Review the rules

Ask:

- What must always be true?
- What cannot happen?
- What status changes are allowed?
- What permissions apply?
- What validation matters?

## 5. Review the examples

Ask:

- Does every important rule have an example?
- Do examples include meaningful failures?
- Are examples concrete enough to review?
- Would a developer and product person interpret the example the same way?

## 6. Review screen content

Ask:

- What data is visible?
- Where does it come from in product terms?
- What action or situation creates it?
- What outside system provides it, if any?
- What happens if it is missing?

## 7. Review the screens

Ask:

- Does each moment have the right UI expectation?
- Are desktop and mobile views considered when they matter?
- Does the screen match the outcome?
- Are required states visible?

## 8. Review readiness

Ask:

- Is the next thing to build obvious?
- Are rules and examples explicit?
- Is important screen content explained?
- Is anything still trapped in vague prose?
- Are any implementation details being mistaken for product intent?

## The final question

Can the team build from this without inventing the important decisions?

If not, the narrative is not ready yet.
