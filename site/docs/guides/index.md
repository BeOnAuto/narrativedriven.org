# Guides

![Five staggered numbered cards: Auto, Review, Design, Spec, System, connected by a dashed path](/images/heroes/guides-index.png){.page-hero}

Use these guides by job. You can practice the method manually, use Auto when the model gets larger, or go deeper into the NDD vocabulary.

## Understand the method

Start here if you are new to NDD.

- [What is NDD?](/what-is-ndd): the problem, the taxonomy, and the basic hierarchy.
- [What Makes a Narrative Buildable?](/what-makes-a-narrative-buildable): the checklist for whether product intent is clear enough to build from.
- [How it Works](/how-it-works): the prompt-to-narrative-to-review loop.

## Use NDD without Auto

Use this path when you want to prove the method with tools you already have.

- [Using NDD without Auto](/using-ndd-without-auto.html): one copyable narrative, one behavior slice.
- [Prompting AI with NDD Vocabulary](/guides/prompting-ai): how to use the taxonomy in prompts.
- [Review a Narrative](/guides/review-a-narrative): a checklist for catching missing outcomes, vague rules, and unexplained data before build.

## Study the example

Use this path when you want proof on a product-sized example.

- [Build the Concert Booking Platform](/guides/build-concert-platform): the canonical example with capacity, waitlists, outcome scenes, and all four moment types.
- [Your First Narrative](/guides/first-narrative): a smaller entry point into the same concert-booking domain.
- [Structuring Domains, Narratives, and Scenes](/guides/structuring-narratives): how to decide what belongs at each level.

## Use Auto

Use Auto when you want the productized workflow: draft narrative generation, visual review, document review, collaboration, and AI-assisted implementation support.

- [NDD in Auto](/guides/narratives-to-code): the product workflow.
- [Running a Collaborative Session](/guides/collaborative-sessions): how teams can work together on the same narrative.
- [Practice NDD in Auto](https://on.auto): open the product.

## Go deeper

When the basics are clear, the explanation and reference sections cover the method in more detail.

- [Reference](/reference/): vocabulary, moment types, rules, examples, and should statements.
- [Explanation](/explanation/): data completeness, cohesion, progressive disclosure, and lineage.
- [For Practitioners](/for-practitioners): how NDD relates to BDD, DDD, EventStorming, TDD, and specification by example.
