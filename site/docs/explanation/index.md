# Explanation

![Hexagon hub with dashed connectors radiating to six concept cards](/images/heroes/explanation-index.png){.page-hero}

Understand the ideas and principles behind Narrative-Driven Development. These pages explain the method language: Domain -> Narrative -> Scene -> Moment, moment types, rules, examples, cohesion, and review practice.

**[Data Completeness](/explanation/data-completeness)**
Important screen content should have an explained source. Nothing important should appear in the product story from nowhere.

**[Cohesion](/explanation/cohesion)**
How goals, scenes, moments, rules, examples, and visible information stay aligned across a narrative.

**[Narrative Review Views](/explanation/one-model-three-views)**
How visual and document reviews help different readers inspect the same product story without exposing implementation details.

**[Progressive Disclosure for Specs](/explanation/progressive-disclosure)**
Why NDD starts with the headline and reveals detail only when it helps the current review decision.

**[Progressive Control](/explanation/progressive-control)**
How NDD stays simple at the start while still allowing deeper product detail when the app gets serious.

**[Standing on Shoulders](/explanation/standing-on-shoulders)**
NDD synthesizes patterns from BDD, EventStorming, Specification by Example, DDD, and User Story Mapping.

**[NDD as a Spec Dialect](/explanation/spec-dialect)**
How NDD works as a taxonomy and method language.

**[The Origin Story](/explanation/origin-story)**
From Xolvio consulting to facilitation workshops to a productized method.
